## 1. Demo 简介

**是什么：** 智家清单是一款面向普通家庭的智能家居决策辅助网站，通过简短的场景化测评（4道题），结合居住情况、家庭成员、核心痛点和平台偏好，为用户生成个性化的「场景包 + 产品清单」推荐方案。

**面向谁：** 想入门智能家居但不知道从哪开始的普通人——租房党、毛坯装修业主、有老人/小孩/宠物的家庭。

**核心功能：**

- **智能测评推荐**：4道题（住房情况 → 房间结构 → 家庭成员 → 场景痛点 + 平台偏好），2分钟完成，自动生成8大场景包分级方案（必需/强烈推荐/可选），支持升级降级
- **已有产品智能去重**：登记家中已有设备后，推荐方案自动排除已拥有产品，实时更新预算
- **场景广场社区**：6+篇真实场景自动化教程（含米家App操作步骤），覆盖安防布防、老人关怀、宠物看护、灯光氛围等

## 2. Demo 创作思路

**灵感来源：** 我自己有一些智能家居的实操经验（免硬装租房改造、全屋灯光自动化、装修预埋规划），发现身边很多人对智能家居感兴趣，但被术语和产品选择劝退。搜索了市面上现有的工具，要么是品牌方的产品推销页，要么是数码博主的长篇评测，没有一个「从需求出发、反推产品清单」的决策工具。

**想解决的问题：**
1. **术语劝退**：用户被"Zigbee""蓝牙Mesh""网关"绕晕，还没开始就想放弃
2. **买错不兼容**：兴冲冲买了设备回家发现协议不匹配、缺网关
3. **以为很贵**：品牌全屋定制动辄两三万，其实租房党几百元就能入门
4. **装修错过预埋**：毛坯房水电阶段不知道该为智能家居预留什么

**为什么做这个方向：** 智能家居的门槛不在产品本身，而在「信息差」。产品已经很便宜了（米家生态），但普通人不知道该买什么、买几个、放哪里、怎么联动。智家清单要做的就是消除这个信息差——用最短的路径，从需求到方案。

## 4. TRAE 实践过程

整个项目从零到完成，全部在 TRAE Work 中完成，经历了以下阶段：

### 阶段一：产品设计与技术选型

与 TRAE 对话确定了产品形态——纯前端静态网站（零部署成本），黑黄主题配色，4道题测评 → 8大场景包推荐的决策闭环。TRAE 帮我输出了完整的产品架构设计，包括数据结构（产品库120+款）、推荐引擎逻辑（5层决策链路：输入→过滤→权重→数量→输出）、路由设计和状态管理方案。

> 🖼️ [截图1] 与 TRAE 讨论产品架构和推荐引擎逻辑的对话

### 阶段二：核心功能开发

TRAE 逐步帮我完成了以下模块：
- **测评系统**：4道场景化题目（住房情况/房间结构/家庭成员/场景痛点+平台），支持分组多选和步进器
- **推荐引擎**：基于用户答案的5层决策链路，平台过滤（米家/HomeKit/华为）+ 租房过滤（剔除需电工设备）+ 权重动态调整 + 按房间数量计算 + 已有产品去重
- **产品数据库**：120+款真实智能家居产品数据，覆盖8大品类，每款产品含评分、安装难度、适用房间、米家自动化指南等
- **场景包系统**：8大场景包（网关/灯光/窗帘/安防/传感/环境/宠物/家务），每个包支持3级升级降级

### 阶段三：体验优化与交互打磨

持续与 TRAE 对话迭代优化：
- 添加「装修预埋建议」卡片（毛坯装修用户可见，涵盖灯具布线/水电预留/网络规划/零线预留）

- 添加「分享方案」和「复制清单」功能（Web Share API + 剪贴板）
- 深色主题文字对比度优化
